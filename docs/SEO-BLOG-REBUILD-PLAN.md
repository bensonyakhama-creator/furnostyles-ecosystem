# Furnostyles Premium SEO Blog Rebuild Plan

**Date:** 2026-05-22  
**Scope:** Blog content architecture, topic prioritisation, and migration strategy  
**Status:** Strategy document — no pages to be built yet

---

## Executive Summary

The old Furnostyles blog contained approximately 100 pages generated from a single thin template. The content was duplicated across articles with only the target keyword swapped. This produced:

- **Zero topical authority** — no article went deep enough to rank for competitive terms
- **Keyword cannibalisation** — multiple URLs targeted identical queries
- **Poor user experience** — readers saw the same generic advice regardless of search intent
- **Wasted crawl budget** — search engines indexed 100+ low-value pages

**The new strategy:** 20–25 genuinely premium articles organised into 5 SEO clusters. Each cluster has 1 cornerstone article (3,000–5,000 words, deep coverage) and 2–4 supporting articles (1,500–2,500 words, specific angles). All articles link vertically within the cluster and horizontally to services, portfolio, and contact pages.

**Projected outcome:**
- 5–8 ranking cornerstone pages within 6 months
- 15–20 first-page rankings for long-tail Kenyan interior / finishing / repair queries within 12 months
- Measurable increase in organic contact-form submissions and service-page entries from blog traffic

## 4. Internal Linking Structure

### Vertical Linking (Within Cluster)

```
Cornerstone (top of cluster)
    ↑
Supporting Articles (link UP to cornerstone)
    ↓
Related Supporting Articles (link ACROSS where relevant)
```

**Rule:** Every supporting article must contain at least 2 contextual links to its cluster cornerstone using varied anchor text.

### Horizontal Linking (Across Clusters)

| From Cluster | Natural Link To | Anchor Example |
|--------------|-----------------|----------------|
| 1 (Interior) | 2 (Materials) | "the flooring materials you choose" |
| 1 (Interior) | 4 (Furniture) | "custom furniture to match your layout" |
| 2 (Finishing) | 3 (Repairs) | "before finishing, fix any underlying damage" |
| 2 (Finishing) | 5 (Real Estate) | "investment-grade finishing for rentals" |
| 3 (Repairs) | 5 (Real Estate) | "maintain rental properties for long-term returns" |
| 4 (Furniture) | 1 (Interior) | "design the room before ordering furniture" |
| 5 (Real Estate) | 2 (Finishing) | "complete house finishing from ceiling to floor" |

### Hub-and-Spoke Linking (Blog → Services)

Every article must link to at least one service page and one conversion page:

| Article Type | Minimum Service Links | Minimum Conversion Links |
|--------------|----------------------|--------------------------|
| Cornerstone | 3 (primary service + 2 related) | 2 (`contact.html` + `portfolio.html`) |
| Supporting | 2 (primary service + 1 related) | 1 (`contact.html` or `services.html`) |

**Anchor text rules:**
- 50% branded/exact ("Furnostyles interior design services")
- 30% partial match ("request a design consultation")
- 20% generic/natural ("our team can help", "get expert advice")

## 5. SEO Cluster Strategy

### Keyword Cannibalisation Prevention

Old problem: 10+ articles targeting "interior design Kenya" with identical content.  
New rule: Only 1 URL per primary keyword group.

| Keyword Group | Owner (Single URL) | Other Articles Must Use |
|---------------|-------------------|------------------------|
| interior design Kenya | Cornerstone 1 | "interior designer in Nairobi", "home styling Kenya" |
| house finishing Kenya | Cornerstone 2 | "construction finishing Nairobi", "shell-to-finish" |
| home repairs Kenya | Cornerstone 3 | "fixing house problems", "repair services Nairobi" |
| custom furniture Kenya | Cornerstone 4 | "bespoke furniture Nairobi", "made-to-order fittings" |
| Airbnb design Kenya | Cornerstone 5 | "rental interior design", "guest house finishing" |

### Content Freshness Schedule

| Article Type | Update Frequency | Trigger |
|--------------|------------------|---------|
| Cornerstone | Quarterly | New regulations, material costs, trend shifts |
| Supporting | Bi-annually | Cost updates, new product availability, case studies |
| All articles | As needed | Broken links, new internal pages, portfolio additions |

### Schema Markup Plan

Every article receives:
- `Article` schema (headline, author, datePublished, dateModified, image)
- `FAQPage` schema (for FAQ sections — enables rich snippets)
- `BreadcrumbList` schema
- Cornerstone articles additionally get:
  - `HowTo` schema (for step-by-step guides)
  - `Review` schema (if client testimonials are included)

## 12. Old Articles to Retire Completely

These articles should not be migrated, merged, or redirected. They offer no unique value and may harm SEO if kept.

| Old Article Pattern | Why Retire | Action |
|---------------------|-----------|--------|
| Articles with <200 words of unique content | Thin content penalty risk | 301 redirect to nearest cornerstone |
| Articles targeting misspelled or nonsensical keywords | No search value | 301 redirect to relevant cornerstone |
| Articles with no internal links to services | Orphan pages with no commercial purpose | 301 redirect to `services.html` |
| Articles that are pure product descriptions (not guides) | Duplicate marketplace content | 301 redirect to `marketplace.html` (when live) or remove |
| Articles with outdated year references (e.g., "2024 trends" that were never updated) | Signals neglect to search engines | Update and merge into a current cornerstone, or remove |
| Articles with broken image paths and no alt text | Poor accessibility and UX | Do not migrate; rebuild from scratch if topic is in the 25 |
| Articles that are exact duplicates of other articles with only the title changed | Canonical / duplicate content issues | 301 redirect to the stronger version |

**Retirement process:**
1. Map each retired URL to the most relevant new cornerstone or supporting article
2. Implement 301 redirects at the server level (or meta refresh if static hosting)
3. Update `sitemap.xml` to remove retired URLs and add new ones
4. Submit updated sitemap to Google Search Console

## 13. Content Quality Standards

Every new article must meet these standards before publication:

| Criterion | Minimum Standard |
|-----------|-----------------|
| Word count | Cornerstone: 3,000+ words. Supporting: 1,500+ words. |
| Original content | 0% duplicate content across articles. Copyscape / original writing only. |
| Images | Minimum 3 unique images per article. Hero image + 2 inline. Proper alt text. |
| Headings | Logical H2 → H3 hierarchy. No skipped levels. Keywords in 50% of H2s naturally. |
| Internal links | Minimum 5 internal links per article (cross-cluster + service + contact). |
| External links | Maximum 2 external links per article. Only to authoritative sources (e.g., NCA Kenya, relevant manufacturers). |
| FAQ section | Minimum 4 questions. Each answer 40–80 words. Uses `<details>` / `<summary>` for interaction. |
| CTA | Minimum 2 CTAs per article. One soft, one hard. |
| Meta description | 150–160 characters. Includes primary keyword and a value proposition. |
| Mobile rendering | Tested on 375px and 768px viewports. No horizontal scroll. Touch-friendly buttons. |
| Page speed | <3s First Contentful Paint on 3G. Images optimised (WebP with JPEG fallback). |
