## 1. PAGE PURPOSES

### index.html (Homepage)
**Purpose:** Brand introduction, service highlights, portfolio preview, and primary conversion points.
**Key Sections:** Hero, stats, services preview, portfolio preview, CTA.
**Primary Goal:** Establish brand credibility and guide users to service pages.

### about.html
**Purpose:** Company story, mission, values, team, and brand history.
**Key Sections:** Hero, about story, mission/vision, team, timeline, CTA.
**Primary Goal:** Build trust and emotional connection with the brand.

### services.html
**Purpose:** Comprehensive service catalog with detailed offerings.
**Key Sections:** Hero, service categories, detailed service cards, process overview, CTA.
**Primary Goal:** Inform visitors about full service capabilities and drive consultation requests.

### repairs.html
**Purpose:** Specialized repair services showcase.
**Key Sections:** Hero, repair categories, before/after showcase, booking CTA.
**Primary Goal:** Drive repair service bookings with clear value proposition.

### portfolio.html
**Purpose:** Project gallery demonstrating craftsmanship and design excellence.
**Key Sections:** Hero, project gallery, case studies, testimonials, CTA.
**Primary Goal:** Showcase quality of work through visual evidence.

### contact.html
**Purpose:** Contact information, inquiry forms, and location details.
**Key Sections:** Hero, contact info, contact form, map placeholder, FAQ, CTA.
**Primary Goal:** Enable easy communication and service inquiries.

### blogs.html
**Purpose:** Content hub for industry insights, design tips, and brand stories.
**Key Sections:** Hero, featured article, article grid, categories, newsletter signup.
**Primary Goal:** Establish thought leadership and SEO value.

#### Neutral Colors
- **White:** #ffffff (Text on dark backgrounds, card backgrounds)
- **Light Grey:** #f8f9fa (Section backgrounds)
- **Medium Grey:** #e9ecef (Borders, dividers)
- **Dark Grey:** #495057 (Secondary text)

#### Status Colors
- **Success:** #28a745
- **Warning:** #ffc107
- **Error:** #dc3545

### Typography System

#### Font Families
- **Headings:** System font stack (San Francisco, Inter, Segoe UI, Roboto)
- **Body:** System font stack (San Francisco, Inter, Segoe UI, Roboto)

- **H1:** 48px (Hero headings)

- **Caption:** 12px (Meta information)

#### Font Weights
- **Bold:** 700 (Headings, emphasis)
- **Semi-Bold:** 600 (Subheadings)
- **Medium:** 500 (Labels)
- **Regular:** 400 (Body text)
- **Light:** 300 (Subtle text)

#### Line Heights
- **Headings:** 1.2
- **Body:** 1.6
- **Tight:** 1.4 (Compact text)

## 4. SPACING SYSTEM

### Base Unit
- **Base:** 8px (All spacing is multiples of 8px)

### Spacing Scale
- **XS:** 8px (0.5rem)
- **SM:** 16px (1rem)
- **MD:** 24px (1.5rem)
- **LG:** 32px (2rem)
- **XL:** 48px (3rem)
- **2XL:** 64px (4rem)
- **3XL:** 96px (6rem)

### Component Spacing
- **Card Padding:** 24px
- **Section Padding:** 96px top/bottom
- **Section Header Margin:** 48px bottom
- **Grid Gap:** 32px
- **Button Padding:** 12px 24px
- **Input Padding:** 12px 16px

### Responsive Spacing
- **Mobile (< 760px):** Reduce section padding to 48px
- **Tablet (760px - 1080px):** Section padding 64px
- **Desktop (> 1080px):** Section padding 96px

**Purpose:** Explain how services work (step-by-step).

**Variants:** Horizontal steps, vertical timeline.

### Team Section
**Purpose:** Introduce team members.
**Components:** Grid layout, team card (photo, name, role, bio).
**Variants:** 2-column, 3-column, 4-column.

## 7. TECHNICAL ARCHITECTURE

### File Structure
```
furnostyles-clean-rebuild/
├── index.html
├── about.html
├── services.html
├── repairs.html
├── portfolio.html
├── contact.html
├── blogs.html
├── assets/
│   ├── css/
│   │   ├── global.css
│   │   ├── layout.css
│   │   ├── components.css
│   │   └── responsive.css
│   ├── js/
│   │   ├── app.js
│   │   └── layout-loader.js
│   └── images/
│       └── [placeholder images]
└── shared/
    └── layout/
        ├── header-data.js
        ├── header-render.js
        ├── footer-data.js
        ├── footer-render.js
        └── footer.css
```

### CSS Architecture
- **global.css:** CSS variables, reset, base typography, utility classes
- **layout.css:** Header, footer, page layout, floating buttons
- **components.css:** Buttons, cards, sections, grids, forms
- **responsive.css:** Mobile breakpoints and responsive adjustments

### JavaScript Architecture
- **app.js:** Global functionality (search, cart placeholder, account placeholder)
- **layout-loader.js:** Header and footer initialization
- **header-data.js:** Header configuration (brand info, navigation, links)
- **header-render.js:** Header rendering logic
- **footer-data.js:** Footer configuration (links, social, contact)
- **footer-render.js:** Footer rendering logic

### Static Hosting Compatibility
- No server-side dependencies
- No build process required
- All files are static HTML/CSS/JS
- Cloudflare Pages, Netlify, GitHub Pages compatible

## 8. SEO FRIENDLY STRUCTURE

### Meta Tags (Per Page)
- Title tag with page-specific content
- Meta description with relevant keywords
- Open Graph tags for social sharing
- Canonical URL
- Structured data (JSON-LD) where applicable

### Semantic HTML
- Proper heading hierarchy (H1 → H2 → H3)
- Semantic elements (header, main, section, article, footer, nav)
- Alt text on all images
- Descriptive link text
- Proper use of ARIA labels for accessibility

### Performance
- Minimal external dependencies
- Optimized image placeholders
- CSS in head, JS at end of body
- Defer non-critical JavaScript
- Lazy loading for images (future enhancement)

## 9. RESPONSIVE BREAKPOINTS

### Breakpoint Strategy
- **Mobile First:** Base styles for mobile, progressive enhancement for larger screens
- **Breakpoints:** 520px, 760px, 1080px

### Responsive Behavior
- **Mobile (< 520px):** Single column, stacked layouts, simplified navigation
- **Small Tablet (520px - 760px):** 2-column grids, adjusted spacing
- **Large Tablet (760px - 1080px):** 3-column grids, medium spacing
- **Desktop (> 1080px):** 4-column grids, full spacing, hover effects
